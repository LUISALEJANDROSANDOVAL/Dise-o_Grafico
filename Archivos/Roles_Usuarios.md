# Roles y Perfiles de Usuario - Proyecto RULEC

El éxito de RULEC radica en su capacidad de adaptar una misma herramienta matemática (la teoría del color) a dos públicos con necesidades y niveles de conocimiento técnico completamente distintos. 

A continuación, se define qué debe cumplir y qué debe poder hacer cada rol dentro de la plataforma.

---

## 1. Perfil: El Microempresario (No Diseñador)

**Objetivo Principal:** 
Obtener una paleta de colores profesional, armónica y justificada para su marca de manera rápida, intuitiva y sin necesidad de conocimientos previos de diseño gráfico.

### ¿Qué debe cumplir la plataforma para este rol?
*   **Fricción Cero / Curva de aprendizaje plana:** No debe sentirse abrumado por números (HEX, RGB) ni términos técnicos (saturación, luminosidad).
*   **Lenguaje Humano:** Los esquemas cromáticos deben tener nombres descriptivos. En lugar de decir "Esquema Complementario Dividido", la interfaz debe decir "Colores de Alto Contraste".
*   **Asistencia Guiada:** Debe contar con un cuestionario inicial (ej. "¿Tu marca es seria o divertida?", "¿A qué rubro perteneces?") que le sugiera un color base de forma automática.
*   **Validación Visual:** Como no sabe de diseño, necesita ver el color aplicado. Debe poder ver su paleta en *mockups* dinámicos (una tarjeta de presentación, un logo falso o una etiqueta) en tiempo real.
*   **Protección contra errores:** El sistema debe actuar como su "Director de Arte", lanzando alertas claras si elige colores que no se leen bien juntos (problemas de contraste).
*   **Educación Justificada:** Debe tener acceso fácil a la sección de "Psicología del Color" para convencerse de que los colores elegidos transmiten lo correcto para su negocio.
*   **Entregable Útil:** Al finalizar, debe poder exportar un PDF (un "Kit Básico de Marca") o un código QR para enviar por WhatsApp a la imprenta, a su socio o a su diseñador web.

---

## 2. Perfil: El Diseñador Gráfico (Junior / Estudiante / Freelancer)

**Objetivo Principal:**
Experimentar con la teoría del color de manera precisa, validar la accesibilidad de sus paletas y acelerar su flujo de trabajo al momento de justificar sus elecciones ante un cliente.

### ¿Qué debe cumplir la plataforma para este rol?
*   **Control Técnico y Preciso:** La interfaz debe liberar todas las restricciones. Deben estar visibles y accesibles los códigos HEX, RGB, CMYK y Pantone.
*   **Ajuste Fino:** Debe tener controles deslizantes (sliders) para modificar sutilmente el matiz (Hue), la saturación (Saturation) y la luminosidad (Lightness).
*   **Herramientas de Accesibilidad:** Debe poder activar simuladores de daltonismo (protanopia, deuteranopia) para asegurar que sus diseños sean inclusivos.
*   **Asistente de Argumentación:** Debe incluir la función "Explicá esta elección". Esta herramienta genera automáticamente un párrafo fundamentado en la teoría del color para que el diseñador lo copie y lo pegue en la presentación para su cliente (ej. "Se utilizó una tríada para generar dinamismo...").
*   **Integración de Flujo de Trabajo:** La exportación debe ser puramente técnica. Debe poder descargar un archivo `.ase` (Adobe Swatch Exchange) para cargarlo directamente en Illustrator o Photoshop, y un archivo con las variables `CSS` o `Tailwind` para entregárselo a un desarrollador web.
*   **Acceso Directo:** Al seleccionar este perfil en la pantalla de inicio, el sistema debe saltarse el cuestionario guiado y llevarlo directamente al círculo cromático interactivo.
